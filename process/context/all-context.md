## How This File Works (the `all-*.md` Convention)

Every `process/context/` directory has one `all-*.md` entrypoint that acts as an attachable quick router for that domain. This root file (`all-context.md`) is the top-level router. Context groups each have their own `all-{group}.md` entrypoint.

**The pattern:**

```
process/context/
  all-context.md                      <-- THIS FILE: root router
  planning/
    all-planning.md                   <-- group router for planning
    example-simple-prd.md             <-- deep doc within the group
    example-complex-prd.md            <-- deep doc within the group
  tests/
    all-tests.md                      <-- group router for tests
    debugging-and-pitfalls.md         <-- deep doc within the group
    e2e-tests.md                      <-- deep doc within the group
  database/
    all-database.md                   <-- group router for database
    schema-guide.md                   <-- deep doc within the group
    migration-procedures.md           <-- deep doc within the group
```

**How agents use it:**

1. Agent reads `all-context.md` first (this file)
2. Finds the relevant context group from the routing tables below
3. Reads that group's `all-{group}.md` entrypoint
4. Only then loads the specific deep doc needed

This layered routing keeps context windows small. Never load the whole `process/context/` tree.

**What each `all-{group}.md` must contain:**

- Scope (what the group covers and does NOT cover)
- Read-when rules (when an agent should load this group)
- Quick procedures or decision rules
- Source paths (list of deeper docs in the group)
- Update triggers (when to refresh this group's content)
- Routing to deeper docs within the group

## Context Group Lifecycle

Context groups are durable knowledge domains, not feature folders.

Create a group when:

- a topic has 3+ durable docs
- a single doc exceeds roughly 800 lines with separable subtopics
- multiple agents repeatedly need only one slice of a large context file
- the topic maps to a stable operational domain (tests, infra, database, auth, UI, workflows, etc.)

Do not create a group when:

- the content is a temporary report
- the content is a plan or execution artifact
- the topic is feature-specific and belongs in `process/features/...`

Move or split one group at a time. Use `all-{group}.md` entrypoints. Run the `audit-context` skill after every context organization change.

## Naming Convention

There are no `README.md` files inside `process/context/`.

Canonical entrypoints use `all-*.md`:

- root: `process/context/all-context.md`
- group: `process/context/{group}/all-{group}.md`

Each `all-{group}.md` file should act as the attachable quick router for that domain:

- tell the agent what the group covers
- give quick procedures and decision rules
- route to smaller deeper files

## Context Update Protocol

When durable project knowledge changes:

1. update the smallest relevant context file
2. update this file if routing, ownership, naming, or groups changed
3. update the owning `all-{group}.md` entrypoint when a group exists
4. run `audit-context`

## Key Patterns and Conventions

**State Management:** Heavy use of in-memory dictionaries (e.g. `USER_CONTEXTS` in `app.py`) for live trading states, cached funds, and active positions.
**Auth Caching:** Fyers API responses are cached in `fyers_client.py` for 5 minutes to prevent rate-limit bans (Error 429).
**Background Workers:** Background loops (`automation_loop`, `market_data_worker`, `trailing_monitor`) run continuously using Python's `asyncio` to manage live trading.
**Database Access:** `models.py` uses `aiosqlite` for asynchronous DB writes to prevent blocking the FastAPI event loop.
**Session Auth (added by security-remediation Phase 1, commit `0e1b78c`):** User sessions use a
signed cookie (`itsdangerous.URLSafeTimedSerializer`, keyed by `SECRET_KEY`) instead of a raw
forgeable `user_id` cookie. `trading-app/auth_utils.py` holds `sign_user_id()` /
`resolve_user_id_from_cookie()` plus login rate-limiting helpers; `app.py`'s
`resolve_authenticated_user_id(request)` is the single shared resolver every route must go
through (do not read `cookies.get("user_id")` directly — that bypasses signature verification).
A 7-day legacy-raw-cookie grace window (`SESSION_MIGRATION_CUTOFF`) is in effect for the
rollout. The Telegram webhook (`app.py` `telegram_webhook`) now requires
`x-telegram-bot-api-secret-token` to match `TELEGRAM_WEBHOOK_SECRET` (fail-closed 403).

**Security Remediation Phase 2 patterns (added by security-remediation Phase 2, commit `c31e950`):**
- **Auth (guest-fallback + OAuth state):** `app.py`'s `get_current_client` now raises `401` when no
  session identity is present (previously silently fell back to a `user_id=0` guest client); the
  only allowlisted unauthenticated route is `GET /api/version`, and a deactivated user's still-valid
  cookie is also rejected with `401`. OAuth `state` is no longer a raw `user_id` — `auth_utils.py`
  adds `generate_oauth_state()` / `consume_oauth_state()`, a signed single-use short-TTL nonce
  checked at `/fyers/callback`. **Known residual (accepted, backlogged):** the nonce is not yet
  bound to the initiating browser/session (PKCE-style verifier cookie) — see
  `process/features/security-remediation/backlog/oauth-state-binding-hardening_NOTE_04-07-26.md`.
- **User lifecycle / background-loop cleanup:** `state.py`'s new `purge_user_runtime()` is the
  shared cleanup called on both deactivate (`toggle_user_status`) and delete
  (`admin_delete_user`, before cascade-delete) — it flips `automation_enabled=False` +
  `hard_exit_triggered=True` on any live `TradingState` and drops the user from
  `USER_CONTEXTS`/`USER_STATES`, so every background loop (`trailing_monitor`, `automation_loop`,
  token-refresh) naturally excludes the user on its next tick. Any new background loop that
  iterates `USER_CONTEXTS` should rely on this cleanup rather than re-checking `is_active` itself.
- **Cascade delete:** `models.py`'s `USER_SCOPED_TABLES` + `delete_user_cascade()` is the single
  place that knows which tables carry a `user_id` column (`user_states`, `daily_pnl_history`,
  `paper_pnl_history`, `system_logs` — NOT `health_memory` or the `swarm_*` tables, which are
  error-/strategy-scoped, not user-scoped). Add new user-scoped tables to `USER_SCOPED_TABLES`
  when introducing them, or deletion will silently leave orphaned rows.
- **LLM auto-remediation guardrail:** `workers/health_agent.py`'s `_match_fix_action()` requires an
  LLM suggestion to exactly equal (trim/lowercase) one of `ALLOWED_FIX_ACTIONS`
  (`restart_ws`/`relogin`/`clear_cache`/`wait`) — substring matches no longer trigger an action.
  `execute_fix()` scopes `restart_ws`/`relogin` to the originating `u_id` when the triggering error
  was user-scoped, falling back to all-users only for genuinely global errors. Residual (accepted,
  documented in the Phase 2 validate-contract Known Gaps): this narrows but does not eliminate LLM
  prompt-injection risk on `execute_fix`.
- **TLS verification:** `fetch_lot_sizes.py` and `app.py`'s `_RSS_SSL_CTX` now build their
  `requests`/`ssl` contexts with `certifi.where()` instead of `verify=False`/`CERT_NONE`. Any new
  outbound HTTPS call in this codebase should default to certifi-backed verification, not disable it.
- **XSS-safe DOM rendering:** `static/landing.html`'s `renderNews()` and `static/admin.html`'s log
  terminals build feed-/log-controlled text via `document.createElement` + `textContent`, never
  `innerHTML`; `landing.html`'s `_safeHttpUrl()` is a reusable `http:`/`https:`-only allowlist
  pattern for any user/feed-controlled URL wired into `window.open`/navigation. Follow this pattern
  for any new UI surface that renders external or attacker-influenceable text.
- **decrypt_val fails loudly:** `models.py`'s `decrypt_val` now raises `DecryptionError` on a
  genuine decryption failure (logged) instead of silently returning ciphertext as plaintext; legacy
  never-encrypted plaintext values still pass through unchanged. Callers must handle the new
  failure signal rather than assuming the return value is always usable plaintext.
